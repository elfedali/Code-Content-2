## Getting Started: Mongo Shell

Welcome to your first hands-on journey with MongoDB! The **Mongo Shell** is your immediate command-line interface to interact with MongoDB databases. It’s the perfect starting point for experimenting, debugging, and building applications without needing a full application server. In this section, we’ll explore the essentials—starting with foundational commands and then diving into the core CRUD operations that power your data workflows. Let’s begin!

### Basic Commands

Before we dive into data operations, let’s master the shell’s environment setup and navigation. These commands help you connect, inspect, and switch contexts within MongoDB.

#### Connecting and Listing Databases
The first step is connecting to your MongoDB instance. When you run `mongo` in your terminal, you’ll enter the shell. If you’re using a specific database host (like `localhost:27017`), you can specify it with:

```bash
mongo localhost:27017
```

Once connected, you can list all available databases with:
```javascript
show dbs
```

This returns a list of databases (e.g., `test`, `admin`, `local`). Note that **`admin`** and **`local`** are system databases and not meant for application data.

#### Switching Databases
To work with a specific database, use the `use` command:
```javascript
use myAppDatabase
```

This switches the current database context for all subsequent operations. Confirm your switch with:
```javascript
db
```

#### Essential Help Commands
The shell is self-documenting! Use these quick commands to navigate:
- `help` → Lists all available commands
- `db` → Shows the current database context
- `db.version()` → Checks your MongoDB version (e.g., `4.4.0`)
- `db.getMongo()` → Shows the underlying MongoDB connection

**Pro Tip**: When stuck, run `help` followed by the command you want (e.g., `help find`) for targeted guidance.

#### Example Workflow
Here’s a practical sequence to get you moving:
```bash
# Connect to MongoDB instance
mongo localhost:27017

# List databases
> show dbs
admin   test   myAppDatabase

# Switch to your app database
> use myAppDatabase
switched to db myAppDatabase

# Verify current context
> db
myAppDatabase
```

This workflow demonstrates how to connect, inspect, and switch contexts—critical for any MongoDB project.

### CRUD Overview

Now that you’re comfortable with the shell environment, let’s explore the **CRUD operations** (Create, Read, Update, Delete) that form the backbone of MongoDB applications. Each operation uses simple, expressive syntax that scales from tiny datasets to complex real-world scenarios.

#### Create (Insert)
Insert new documents into a collection using `insertOne` or `insertMany`. Collections are like "tables" in relational databases—each holds documents (JSON-like objects).

**Example**: Create a `users` collection with one document:
```javascript
db.users.insertOne({ name: "Alice", email: "alice@example.com" })
```

**Output**:  
`{ "insertedId" : "ObjectId(...)" }` (a unique `_id` generated by MongoDB)

**Multiple inserts**:  
```javascript
db.users.insertMany([
  { name: "Bob", email: "bob@example.com" },
  { name: "Charlie", email: "charlie@example.com" }
])
```

#### Read (Query)
Retrieve documents with `find()`. By default, it returns all documents in a collection. Use filters to narrow results.

**Basic query**:  
```javascript
db.users.find()
```

**Filtered query** (find users with email containing `@example.com`):  
```javascript
db.users.find({ email: { $regex: "example.com" } })
```

**Projection** (return only `name` field):  
```javascript
db.users.find({}, { name: 1, email: 0 })
```

#### Update (Modify)
Update existing documents with `updateOne` (single document) or `updateMany` (multiple documents).

**Update one user's email**:  
```javascript
db.users.updateOne(
  { name: "Alice" },
  { $set: { email: "alice.new@example.com" } }
)
```

**Update multiple users**:  
```javascript
db.users.updateMany(
  { email: { $regex: "old@example.com" } },
  { $set: { email: "$regex" } } // Example: add domain
)
```

#### Delete (Remove)
Remove documents with `deleteOne` (single) or `deleteMany` (multiple).

**Delete one user**:  
```javascript
db.users.deleteOne({ name: "Bob" })
```

**Delete all users**:  
```javascript
db.users.deleteMany({})
```

#### CRUD Comparison Table
| Operation | Command             | Key Use Case                          | Example Snippet                                  |
|-----------|---------------------|----------------------------------------|--------------------------------------------------|
| **Create** | `insertOne`/`insertMany` | Adding new data                       | `db.users.insertOne({ name: "Alice" })`           |
| **Read**   | `find()`             | Retrieving data                       | `db.users.find({ email: "alice@example.com" })`   |
| **Update** | `updateOne`/`updateMany` | Modifying existing data               | `db.users.updateOne({ name: "Alice" }, { $set: { email: "new@example.com" } })` |
| **Delete** | `deleteOne`/`deleteMany` | Removing data                        | `db.users.deleteOne({ name: "Bob" })`             |

#### Why This Matters
These operations are **non-blocking** and **asynchronous** by default—meaning you can chain them without waiting for responses. For example:
```javascript
// Insert, then read the new document
db.users.insertOne({ name: "David" })
db.users.find({ name: "David" })
```

This flexibility lets you build responsive applications where data operations happen in the background without disrupting user experience.

## Summary

The Mongo Shell is your immediate, powerful gateway to MongoDB. By mastering **basic commands** like `show dbs`, `use`, and `db`, you can quickly navigate your database environment. Then, with **CRUD operations**—especially `insertOne`, `find`, `updateOne`, and `deleteOne`—you’ll handle data workflows with precision and ease. Start small: experiment with one command at a time, and watch your confidence grow. Remember: the shell is your most intuitive tool for rapid iteration. 🐍
